樹狀結構與二元樹：階層數據的完美呈現

從家譜到搜尋引擎

樹狀結構是一種非線性的資料結構，它以分層的方式組織數據，非常適合表示具有層次關係的數據，例如文件系統、組織架構或家譜。在電腦科學中，樹被廣泛應用於資料庫索引、編譯器設計和網路路由等領域。

樹的基本概念

  • 節點 (Node)：樹的基本組成單位，包含數據和指向其子節點的指針。
  • 根節點 (Root Node)：樹的最頂端節點，沒有父節點。
  • 父節點 (Parent Node)：直接位於某節點上方的節點。
  • 子節點 (Child Node)：直接位於某節點下方的節點。
  • 葉節點 (Leaf Node)：沒有子節點的節點。
  • 邊 (Edge)：連接兩個節點的線。
  • 深度 (Depth)：從根節點到某節點的路徑長度。
  • 高度 (Height)：從某節點到其最遠葉節點的最長路徑長度。

二元樹 (Binary Tree)

二元樹是樹狀結構中最常見的一種，它的每個節點最多只有兩個子節點，通常稱為左子節點和右子節點。

  • 二元搜尋樹 (Binary Search Tree, BST)：

    一種特殊的二元樹，對於任意節點，其左子樹中所有節點的值都小於該節點的值，而右子樹中所有節點的值都大於該節點的值。這使得二元搜尋樹非常適合進行高效的搜尋、插入和刪除操作。

  • 平衡二元樹 (Balanced Binary Tree)：

    為了避免二元搜尋樹在最壞情況下退化成鏈結串列（導致效率降低），平衡二元樹會確保樹的高度保持在一定範圍內，例如 AVL 樹、紅黑樹。

二元樹的遍歷 (Traversal)

遍歷是指按照特定順序訪問樹中所有節點的過程。常見的二元樹遍歷方式有：

  • 前序遍歷 (Pre-order Traversal)：根節點 -> 左子樹 -> 右子樹。
  • 中序遍歷 (In-order Traversal)：左子樹 -> 根節點 -> 右子樹。對於二元搜尋樹，中序遍歷會得到一個排序好的序列。
  • 後序遍歷 (Post-order Traversal)：左子樹 -> 右子樹 -> 根節點。
  • 層次遍歷 (Level-order Traversal)：從上到下，從左到右，逐層訪問節點。

應用場景

  • 資料庫索引：B 樹和 B+ 樹是資料庫中常用的索引結構，能快速檢索數據。
  • 檔案系統：目錄和子目錄的組織方式。
  • 編譯器：語法樹 (Parse Tree) 用於表示程式碼的結構。
  • 決策樹：機器學習中用於分類和回歸的模型。
